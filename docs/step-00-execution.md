# Step 0 실행 산출물

## 목적
`docs/steps/step-00.md`의 3개 항목(정책 준수, 파이프라인 분리, 핵심 병목)을 실제 운영 기준으로 고정한다.

## 0.1 ToS/정책 준수 가드레일
- 공식 Brain API를 기본 경로로 사용한다.
- 크롤링/복사는 아래 조건을 모두 만족할 때만 예외적으로 검토한다.
  - API로 동일 정보를 얻을 수 없다.
  - 약관/robots/접근 정책 위반이 없다.
  - 요청 부하를 최소화하고 인증/보안 우회를 하지 않는다.
- 금지 사항:
  - 인증 우회, CAPTCHA/보안 우회, 과도한 병렬 요청, 비인가 데이터 복제

## 0.2 플랫폼용/실거래용 파이프라인 분리 원칙
- 플랫폼용 알파 생산 파이프라인:
  - 목표: BRAIN 제출 지표(Sharpe/Fitness/Turnover 등) 개선
  - 데이터/평가 기준: 플랫폼 제공 데이터와 지표
- 실거래용 리서치 파이프라인:
  - 목표: 체결 가능성 포함한 PnL 안정성 검증
  - 추가 요구: 슬리피지/거래비용/리밸런싱/리스크 모델
- 강제 규칙:
  - 두 파이프라인의 결과를 동일 성과로 해석하지 않는다.
  - 보고서/로그에서 파이프라인 타입을 반드시 태깅한다.

## 0.3 자동화 핵심 병목 3개 운영 지표
1. 메타데이터 동기화 품질
- 지표: 동기화 성공률, 최신화 시각, 누락 필드 비율
- 점검 주기: 일 1회

2. FastExpr 정적 검증 품질
- 지표: 시뮬 제출 전 검증 실패율, 제출 후 API 문법 에러율
- 점검 주기: 배치 실행마다

3. 평가/피드백 루프 학습성
- 지표: 반복 라운드별 통과율 개선, 동일 실패 재발 비율
- 점검 주기: 주 1회
