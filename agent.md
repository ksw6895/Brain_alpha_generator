# Meta Prompt For Next Agent

다음 작업자는 아래 절차를 반드시 따른다.

## 1) 시작 전 확인
- `docs/steps/README.md`를 먼저 읽고 현재 목표 step 문서를 확인한다.
- `docs/baseguideline.md`를 읽고 정책 준수/파이프라인 분리 원칙을 전제로 작업한다.
- 요청받지 않은 step은 수행하지 않는다.

## 2) 단계 수행 원칙
- 현재 step 문서의 체크리스트 항목만 처리한다.
- 산출물은 `docs/` 하위에 재현 가능한 형태로 남긴다.
- 완료한 항목만 `[x]`로 표시하고, 미수행 항목은 `[ ]`로 유지한다.
- step 완료 시 의존성과 다음 step 진입 조건을 1~3줄로 남긴다.

## 3) 코딩 외 작업 기록 의무
- 코드 변경이 없는 업무라도 반드시 `docs/non-coding-tasks.md`에 기록한다.
- 새 항목 작성 시 `ID`, `작업`, `상태`, `담당`, `증빙/링크`를 채운다.
- 상태 전환 규칙:
  - 시작 전: `TODO`
  - 진행 중: `DOING`
  - 완료: `DONE` + 완료일
  - 막힘: `BLOCKED` + 막힌 이유를 작업명에 괄호로 추가

## 4) 금지 사항
- ToS/정책 위반 가능성이 있는 수집/자동화를 임의로 추가하지 않는다.
- 플랫폼용 지표와 실거래 가능성을 동일 근거로 혼합 보고하지 않는다.
- 체크리스트를 실제 수행 없이 완료 처리하지 않는다.
